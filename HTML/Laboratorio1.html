Muchas veces nos preguntamos cual es la diferencia entre Internet y la World Wide Web y bueno esto es un poco de sus diferencias. El internet existió antes que la Web. El internet es una grande red de computadoras conectas entre sí. Y la World Wide Web es una grande colección de páginas que se llevan a cabo en una red de computadoras. Cuando se navega a través de diferentes dispositivos se utiliza el internet para poder acceder a la web. El problema que antes existía era que no había forma de poder realizar una conexión con todos los documentos y el internet. En analogias las carreteras que conectan a muchos paises y ciudades seria el Internet y la web serian todas las páginas que se encuentran en ellas las cuales son alojadas en servidores. La web es la red que esta corriendo sobre el internet.

El propósito de los métodos de petición es para poder determinar la acción que se desea realizar para un recurso determinado.

  • GET: solicita una representación de un recurso específico. Estas peticiones solo recuperan datos. Este es utilizado para consultar información al servidor. Cuando se accede a una página.
  • HEAD: pide una respuesta igual a que la petición GET, la difeerncia es que este lo hace sin el cuerpo de respuesta. Para obtener información sobre un determinado recurso sin necesidad de regresar el registro.
  • POST: se utiliza para enviar una entidad a un recurso en específico, el cual genera un cambio en el estado o efectos secundarios en el servidor. Utilizado para solicitar la creación de un nuevo registro algo similar a realizar un insert en la base de datos.
  • PUT: remplaza las representaciones actuales del recurso de destino con la carga útil de la petición. Actualizar por completo un registro existente parecido a un update a la base de datos.
  • PATCH: se utiliza para poder aplicar modificaciones a un recurso. Similar al put, pero se utiliza cuando solo hay que actualizar un fragmento del registro y no todo en su totalidad.
  • DELETE: borra un recurso en específico. Utilizado cuando hay que eliminar un registro existente, parecido al delete en la base de datos.

Usualmente para poder enviar un formulario HTML se puede utilizar los métodos get y post sin embargo en el caso específico de ingresar usuario y contraseña se utiliza el post. Debido a que la si usamos el método POST en la URL de la página de destino se muestran los datos enviados,para información más sensible como usuarios, contraseñas, correos electrónicos se utiliza el método POST.

Se utiliza el método GET cuando a través de un navegador web se accede a una página a través de un URL.

Lo que ocurre cuando un servidor web devuelve una respuesta HTTP con código 200 significa que la solicitud ha tenido éxito. Dependiendo del método http utilizado es lo que nos refermimos a éxito.

  • GET: el recurso se ha obtenido y se transmite en el cuerpo del mensaje.
  • HEAD: los encabezados de entidad están en el cuerpo del mensaje
  • PUT o POST: el recurso que describe el resultado de la acción se transmite en el cuerpo del mensaje.

Cuando ocurre un error 404 no es reposnabilidad del desarrollador es culpa del usuario ya que se metio a una página rota y en algunas veces los desarrolladores usan este código para poder defenderse de cualquier usuario intentando alterar los datos.

Cuando ocurre un error 500 no es responsabilidad del desarrollador de corregir el sitio web ya que este error está relacionado con el servidor, lo que ocurre es que el servidor ha encontrado una situación que no sabe cómo llevar a cabo. Esto quiere decir que se espera que el problema sea solucionado por parte del desarrollador y administrador del servidor sin embargo no del desarrollador de la página.

Cuando un atributo HTML5 esté depreciado o desaprobado quiere decir que son partes del estándar HTML cuyo uso no es sugerido. Esto se debe a que el estándar es actualizado y atributos son agregados, eliminados o desaprobados. Se queda atrás debido al surgimiento de nuevas estructuras. La decisión de seguir usando atributos desaprobados depende del autor, y muchos fabricantes de navegadores todavía dan soporte para algunos atributos desaprobados sin embargo se alienta al desarrollador a encontrar otras formas de poder lograr los efectos y atributos desaprobados.

Estas son unas de las principales diferencias entre HTML 4 y HTML 5:

La Sintaxis, incluye un control de errores para que la sintaxis pueda ser compatible con las implementaciones más populares. Se puede utilizar XHTML5.

Codificación de Caracteres, capaz de definir el charset del documento.

DOCTYPE, se requiere de este elemento el cual es declarado al principio de la pagina, con lo que se asegura que el navegador renderiza la página en modo estándar.

Modelos estrictos de contenidos, como toda actualización nuevo elementos como section, article, aside, header.

Las tablas son construidas utilizando elementos table para el contenido principal, tr representando las filas, td representando a las celdas. Los elementos estructurales básicos de las tablas son, encabezados de las tablas, celdas de datos de las tablas y títulos de las tablas. Para poder tener celdas de encabezado se declara th. Para el estilo de la tabla podemos hablar de los siguientes componentes, ancho de tabla y celda, alineación del texto, bordes, relleno, fondo, líneas de celda etc. Las cuales podemos lograr a través de un CSS.

Los principales controles de una forma HTML 5 pueden ser agrupados en base a su función:

  • El mas importante es el !DOCTYPE declarado a principio de la página.
  • De los metadartos del documento tenemos head, title, style.
  • Para las secciones tenemos body, section, article, h1(headers), footer.
  • Para agrupar contenido tenemos li listas, ul atriculos sin orden, p párrafo.

Acerca del ciclo de vida y desarrollo de los sistemas de información tenemos lo siguiente.
Todos los procesos que pasan desde que se crea la necesidad de un sistema hasta que otro lo sustituya pertenece al ciclo de vida. Existen ciertas etapas para el ciclo de vida y del desarrollo de estos las cuales son:

  • Planificación: se determina el ámbito del proyecto, se hace un estudio de viabilidad, se analizan riesgos, su costo y determinar los recursos del proyecto.
  • Análisis: se establece que es lo que va a realizar el sistema de información, esta etapa corresponde al proceso en el que se descubre que es lo que se necesita y de igual forma los requerimientos del sistema.
  • Diseño: se representan las caracteristicas del sistema que permitira implementar de forma efectiva todos requisitos desde el punto de vista del usuario.
  • Implementación: seleccionamos las herramientas adecuadas, el entorno de desarrollo y el lenguaje de programacion adeacuado para poder llevar a cabo el programa.
  • Pruebas: se detectan los errores que se hayan podido cometer en cualquier de las etapas anteriores.
  • Instalación: en esta etapa se pone el sistema en funcionamiento.
  • Uso y Mantenimiento: siendo la etapa mas importante del desarrollo, se propone eliminar defectos a lo largo de la vida del programa, adpatarlo a nuevas necesidades y añadir funcionalidad nueva que se vaya presentando a lo largo de la vida del mismo.
